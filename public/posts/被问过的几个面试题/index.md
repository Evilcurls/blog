# 被问过的几个面试题



##  1.当矩阵是百万级规模时，如何用概率论快速判断 \(AB = C\)？

在大规模矩阵计算的场景中（比如 \(n = 10^6\)），直接计算矩阵乘积 AB 再与 C 比较是完全不现实的 —— 毕竟 \(n^3\) 的复杂度会让超级计算机都望而却步。这时候，**概率论方法**就能帮我们在可接受的时间内，以极高的概率完成判断。

当 \(n = 10^6\) 时，直接计算两个 \(n \times n\) 矩阵的乘积 AB，需要执行 \(n^3 = 10^{18}\) 次运算。哪怕是当前最快的超级计算机（每秒约 \(10^{17}\) 次运算），也需要近 10 秒才能完成一次这样的乘法，更别说矩阵存储（单矩阵就需要 \(10^{12}\) 个元素，约 4TB 空间）的压力了。

所以，直接计算 AB 再和 C 逐元素比对，在工程上是**完全不可行**的。

**方法：随机向量检测**

我们利用一个关键的数学逻辑：

- 如果 \(AB = C\)，那么对**任意**向量 都有 \(AB\mathbf{x} = C\mathbf{x}\)；
- 如果 \(AB \neq C\)，那么矩阵 \(D = AB - C\) 是**非零矩阵**，但它的 “零空间”（满足 \(D\mathbf{x} = \mathbf{0}\) 的向量 \(\mathbf{x}\)）是**低维的**，随机选到这样的 \(\mathbf{x}\) 的概率极低。

基于这个逻辑，我们可以设计如下步骤：

**步骤 1：生成随机向量 x**

构造一个 n 维的随机向量  x，比如让每个分量独立服从伯努利分布（取 0 或 1），或标准正态分布 n to (0,1)\)。

**步骤 2：分步计算 (ABx) 和 \(Cx)**

这里的关键是**利用矩阵 - 向量乘法的结合律**，避免直接计算 AB：

- 先算 \(\mathbf{y} = B\mathbf{x}\)（矩阵 B 与向量 \(\mathbf{x}\) 相乘，复杂度 \(O(n^2)\)）；
- 再算 \(AB\mathbf{x} = A\mathbf{y}\)（矩阵 A 与向量 \(\mathbf{y}\) 相乘，复杂度 \(O(n^2)\)）；
- 同时计算 \(C\mathbf{x}\)（矩阵 C 与向量 \(\mathbf{x}\) 相乘，复杂度 \(O(n^2)\)）。

**步骤 3：比较并验证**

- 如果 ABx  不等于  Cx，**可以 100% 确定** AB 不等于 C；
- 如果 ABx=Cx**不能 100% 确定** \(AB = C\)，但可以通过**多次重复检验**（每次换不同的随机向量 x），把 “误判概率” 降到极低。



**方法优点**

1. 计算复杂度的飞跃

直接计算 AB 是 \(O(n^3)\)，而用随机向量检验每次仅需 \(O(n^2)\)（甚至在矩阵稀疏时，复杂度可低至 \(O(n)\)）。当 \(n = 10^6\) 时，\(n^2 = 10^{12}\)，比 \(n^3\) 小了 1000 倍，工程上完全可执行。

2. 可靠性的数学保证

对于非零矩阵 \(D = AB - C\)，其零空间的维度最多是 \(n-1\)（远小于 n 维的全空间）。如果 x是随机生成的（比如服从连续分布），那么 x 落入零空间的概率是**0**；即使是离散分布，这个概率也会随着检验次数的增加指数级下降（比如每次误判概率 1/2，检验 10 次后误判概率就小于 \(1/1000\)）。



## 2.简述中心极限定理

简单来说，中心极限定理讲的是：**如果有一堆独立的随机因素在共同影响一个结果，不管这些因素本身是什么分布，只要它们的影响都不算特别极端，那么这个结果的整体分布最终会趋近于正态分布（也就是钟形曲线）**。

举个例子，假设你要统计一个城市所有人的身高。每个人的身高受到遗传、饮食、运动等很多独立因素的影响。有的因素让身高高一点，有的让身高矮一点。当把几十万人的身高数据汇总后，你会发现这些数据的分布会呈现出中间高、两边低的钟形 —— 这就是中心极限定理在起作用。

再比如抛硬币，单次抛硬币是正面还是反面是随机的，但如果抛很多次（比如 1000 次），记录每次正面出现的次数，你会发现这些次数的分布也会趋近于正态分布。

中心极限定理的价值在于，它让我们在面对复杂的随机现象时，有了一个可预测的 “终点”。不管底层的随机因素多复杂，只要它们相互独立、影响均匀，最终的结果分布都会向正态靠拢。这也是统计学中很多方法（比如假设检验、置信区间）的重要理论基础。



## 3.如何解决机器学习中类别不平衡的问题

在机器学习中，“类别不平衡” 是个很常见的问题。比如在诈骗检测中，正常交易是绝大多数，诈骗交易寥寥无几；在疾病筛查中，健康人群远多于患病人群。如果直接用这样的数据训练模型，模型很容易 “偏向” 多数类，对少数类的预测效果极差。那该怎么处理呢？

### 1. 数据层面：调整样本分布

- **过采样**：对少数类样本进行复制，增加其数量。比如少数类只有 100 个样本，多数类有 1000 个，我们可以把少数类样本复制几次，让它的数量接近多数类。不过要注意，简单复制容易导致过拟合，现在常用的是 “合成少数类过采样技术（SMOTE）”，它会根据少数类样本的特征，合成新的、合理的样本。
- **欠采样**：对多数类样本进行删减，减少其数量。比如从多数类的 1000 个样本中随机选 300 个，和少数类的 100 个样本一起训练。这样能平衡数据量，但可能会丢失多数类的一些重要信息。

### 2. 模型层面：给少数类 “加权”

在训练模型时，我们可以给少数类样本设置更高的权重。比如在损失函数中，少数类样本预测错误时，模型会受到更严重的惩罚。这样模型就会更 “关注” 少数类，提高对它的预测准确率。像决策树、随机森林等模型都支持设置样本权重。

### 3. 评估层面：换个指标看效果

传统的准确率指标在类别不平衡时会失效（比如多数类占 90%，模型全猜多数类也能有 90% 准确率）。这时候可以看**精确率、召回率、F1 分数**这些指标。

- 精确率：预测为少数类的样本中，真正是少数类的比例；
- 召回率：真正的少数类样本中，被模型预测出来的比例；
- F1 分数：精确率和召回率的综合，能更全面地反映模型对少数类的预测能力。

### 4. 算法层面：专门的不平衡算法

有些算法本身就对类别不平衡有较好的适应性，比如**集成学习中的极端随机森林、梯度提升树（GBDT）** 等，它们可以通过组合多个弱分类器，减少对多数类的偏向。另外，“异常检测” 思路也常被用于少数类占比极低的场景，把少数类当作 “异常” 来识别。



## 4. 递归求斐波那契数列的时间和空间复杂度

**时间复杂度**

定义 T(n) 为计算 fib(n) 所需的总计算量。

根据递归函数的定义：
fib(n) = fib(n-1) + fib(n-2)

我们可以得出计算量的关系式：
T(n) = T(n-1) + T(n-2) + 1 (最后的 +1 代表那一次加法操作)

为了简化分析，我们可以忽略 +1，因为我们只关心增长的趋势。
T(n) ≈ T(n-1) + T(n-2)

我们再看 T(n-1):
T(n-1) ≈ T(n-2) + T(n-3)

代入进去：
T(n) ≈ [T(n-2) + T(n-3)] + T(n-2) = 2 * T(n-2) + T(n-3)

T(n) 大约是 2 * T(n-2)。这意味着，n 每减少2，计算量就要翻一倍。这是一种典型的**指数增长**模式。

所以时间复杂度为2^n

## 5.矩阵中的秩代表什么

### 

矩阵的秩是矩阵中行（或列）向量组的极大线性无关组所含向量的个数，它反映了矩阵的线性相关性和空间维度特性。例如，对于一个 m x n的矩阵，其秩 r 满足 0 < r <min (m,n)。秩为 r 意味着矩阵中存在 r 个线性无关的行（或列），其余行（或列）都可以由这 r 个行（或列）线性表示。



## 6.归并排序和快速排序

- 归并排序
  - 思想：分治法，将数组不断分成两半，分别排序后再合并。
  - 时间复杂度：\(O(n\log n)\)（最好、最坏、平均均是）。
  - 空间复杂度：\(O(n)\)（需要额外空间存储合并后的数组）。
  - 稳定性：稳定（相等元素的相对顺序在排序后不变）。

- 快速排序
  - 思想：分治法，选择一个 pivot（基准元素），将数组分成小于 pivot 和大于 pivot 的两部分，递归排序这两部分。
  - 时间复杂度：平均 \(O(n\log n)\)，最坏 \(O(n^2)\)（如 pivot 选择不当，数组已完全有序时）。
  - 空间复杂度：\(O(\log n)\)（递归栈的深度）。
  - 稳定性：不稳定（pivot 交换时可能改变相等元素的相对顺序）
